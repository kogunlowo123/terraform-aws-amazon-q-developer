Executive Summary

This module provisions a complete AI-powered developer productivity platform on AWS. It creates a CI/CD pipeline with CodePipeline and CodeBuild, integrates Amazon CodeGuru Reviewer for automated code reviews, enables security scanning (Bandit, Safety), manages packages via CodeArtifact, and provides encrypted artifact storage with SNS-based notifications. All resources are encrypted with KMS and follow least-privilege IAM practices.

Component Breakdown

CI/CD Pipeline

  • CodePipeline -- Orchestrates Source, Build, and custom stages
  • CodeBuild -- Executes builds with Python 3.12 / Node 20 runtime
  • CodeStar Connection -- Links to GitHub, GitLab, or Bitbucket
  • BuildSpec -- Inline spec with install, pre_build, build, post_build phases

Code Quality

  • CodeGuru Reviewer -- Automated code reviews (optional, default on)
  • Security Scanning -- Bandit + Safety scans during pre_build (optional)

Package Management

  • CodeArtifact Domain -- Encrypted package domain
  • CodeArtifact Repository -- Project packages with npm upstream proxy

Notifications

  • SNS Topic -- KMS-encrypted notification channel
  • Notification Rules -- Pipeline and build event subscriptions
  • Email Subscriptions -- Configurable via notification_emails

Storage & Encryption

  • S3 Bucket -- Versioned, encrypted artifact store (90-day lifecycle)
  • KMS Key -- Dedicated key with automatic rotation (or bring your own)

Data Flow

  1. Developer pushes code to source repository (GitHub/GitLab/Bitbucket)
  2. CodeStar Connection detects change and triggers CodePipeline
  3. Source Stage pulls code through the connection into S3 artifacts
  4. Build Stage invokes CodeBuild which runs the inline BuildSpec
  5. BuildSpec installs deps, runs security scans (Bandit, Safety), builds project, and runs tests
  6. Build artifacts are stored in the KMS-encrypted S3 bucket
  7. CodeGuru Reviewer independently reviews code and posts findings
  8. Pipeline/build events fire notification rules to SNS, which delivers to email subscribers
  9. Optional custom stages (Deploy, Approval, etc.) execute after Build

Industry Adaptation

Software Engineering Teams

Full CI/CD with automated code quality reviews. Extend pipeline_stages for multi-environment deployments (staging, production). Enable CodeArtifact for private package management.

Security-Conscious Organizations

Enable security_scanning for SAST/SCA in every build. Use CodeGuru Reviewer for ML-powered vulnerability detection. All artifacts encrypted with customer-managed KMS keys.

Enterprise / Multi-Repo

Deploy one module instance per repository. Share a single CodeArtifact domain across projects. Use SNS notifications for centralized build monitoring.

Startups / Fast Iteration

Minimal config: just project_name and repository_url. Automatic builds on every push. Add custom stages as the deployment process matures.

Production Readiness Checklist

  • Complete CodeStar Connection handshake in AWS Console after terraform apply
  • Verify KMS key policy allows intended principals
  • Subscribe security/ops email addresses to SNS topic
  • Confirm build image supports your language runtimes
  • Test pipeline with a manual trigger: aws codepipeline start-pipeline-execution
  • Review CodeGuru Reviewer findings for initial recommendations
  • Enable S3 access logging on the artifact bucket for auditing
  • Configure remote Terraform state backend (S3 + DynamoDB)
  • Set up cross-account deployment stages if multi-account
  • Tag all resources with Environment, Team, ManagedBy
